# Quick Task 260602-001: Fix brew install — goreleaser archive missing _docker-deploy at root

## Problem

`brew install docker-deploy` fails with:
```
Errno::ENOENT: No such file or directory - _docker-deploy
```

The goreleaser archive config includes `contrib/_docker-deploy` without `strip_parent: true`,
so the file is packed as `contrib/_docker-deploy` inside the tarball. The Homebrew formula
install stanza does `install "_docker-deploy"` expecting it at the archive root — hence ENOENT.

## Tasks

### Task 1: Fix goreleaser archive files config

**Files:** `.goreleaser.yaml`
**Action:** Add `strip_parent: true` to both `contrib/_docker-deploy` and `contrib/docker-deploy.bash` entries
**Verify:** Files section uses `strip_parent: true` for both contrib entries
**Done:** When `.goreleaser.yaml` has `strip_parent: true` on both file entries

## Root cause

goreleaser v2 preserves directory structure by default in archives. `strip_parent: true` tells
goreleaser to drop the `contrib/` prefix so files land at tarball root, matching the brew formula.
