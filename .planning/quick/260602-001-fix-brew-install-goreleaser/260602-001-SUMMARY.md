# Quick Task 260602-001: Fix brew install — goreleaser archive missing _docker-deploy at root

## What was done

Added `strip_parent: true` to both `contrib/_docker-deploy` and `contrib/docker-deploy.bash`
entries in `.goreleaser.yaml`.

**Root cause:** goreleaser v2 preserves directory structure by default. Without `strip_parent: true`,
both files were packed as `contrib/_docker-deploy` and `contrib/docker-deploy.bash` inside the
tarball. The Homebrew formula install stanza uses `install "_docker-deploy"` and
`install "docker-deploy.bash"` which expect the files at the archive root — hence ENOENT on install.

**Fix is in the main repo** (not the homebrew-docker-deploy formula). The formula is correct;
goreleaser was responsible for placing the files at the right path in the release archive.

## Commits

- fix(release): strip contrib/ prefix from completion files in goreleaser archive

## Verification

Next release will include `_docker-deploy` and `docker-deploy.bash` at the root of the tarball.
Brew formula install will succeed without changes.
