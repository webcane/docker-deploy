# Quick Task 260519-q02: Fix health check docker inspect exit status 1

## Root Cause

`inspectHealth()` runs:
```
docker inspect --format '{{.State.Health.Status}}' <container>
```

When a container has **no HEALTHCHECK instruction** in its Dockerfile, `.State.Health` is `null` in the inspect JSON. The Go template engine then fails trying to access `.Status` on a nil pointer, causing `docker inspect` to exit with code 1.

The existing error handler treats inspect failures as "unknown, continue polling" — so the container is never marked done and the health check times out after 30s.

## Fix

Change the format template to handle nil Health:

```go
// internal/health/poll.go: inspectHealth()
cmd := "docker inspect --format '{{if .State.Health}}{{.State.Health.Status}}{{else}}none{{end}}' " + filetransfer.ShellQuote(containerName)
```

This returns:
- `"healthy"` / `"unhealthy"` / `"starting"` for containers with HEALTHCHECK
- `"none"` for containers without HEALTHCHECK

The existing switch in `pollContainers` already handles `""` and `"none"` as pass-through (no HEALTHCHECK defined → mark done and warn). No other logic changes needed.
