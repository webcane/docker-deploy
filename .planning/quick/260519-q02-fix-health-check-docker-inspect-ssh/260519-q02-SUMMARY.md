# Quick Task 260519-q02: Fix health check container inspection

## What happened

**Root cause:** `docker inspect --format '{{.State.Health.Status}}'` exits with status 1 when a
container has no HEALTHCHECK instruction — because `.State.Health` is `null` in the JSON and the
Go template engine fails accessing `.Status` on a nil pointer.

The error handler in `pollContainers` treated inspect failures as "unknown, continue polling",
so the container was never marked done. Health check always timed out after 30s.

**Fix:** Changed the format template in `inspectHealth()`:

```go
// Before (exits 1 for containers with no HEALTHCHECK):
cmd := "docker inspect --format '{{.State.Health.Status}}' " + filetransfer.ShellQuote(containerName)

// After (returns "none" for containers with no HEALTHCHECK):
cmd := "docker inspect --format '{{if .State.Health}}{{.State.Health.Status}}{{else}}none{{end}}' " + filetransfer.ShellQuote(containerName)
```

The existing `"none"` branch in `pollContainers` already marks such containers as done with a
warning: `"Warning: container X has no HEALTHCHECK defined"`. No other logic changes required.
