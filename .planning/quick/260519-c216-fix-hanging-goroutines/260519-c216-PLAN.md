# Fix Hanging Goroutines in RunCompose Test

## Problem

Test goroutines hang when executing RunCompose. Root cause:
- Mock SSH server's TCPListener remains open, still accepting connections
- SSH client goroutines wait for connection closure
- Test context never signals completion to internal goroutines

Goroutine stack:
- Goroutine 5 (test): `run.go:154` → `wg.Wait()` blocked
- Goroutine 6: Mock server → `TCPListener.Accept()` blocked
- Goroutine 7: SSH handling → reading packets, no closure signal
- Goroutines 33-34: SSH client → waiting for data/closure

## Solution

Modify `startMockComposeSSHServer` to return a closer function. Test calls `defer closeServer()` to clean up after assertions. This is more reliable than context cancellation or modifying RunCompose's signature.

## Rationale

- **Why not context cancellation?** Requires changes to RunCompose's internal select loops; less reliable
- **Why not modify RunCompose?** Would require RunCompose to know when test is done; breaks encapsulation
- **Why closer function?** Cleanest: test owns resource, test cleans up. No signature changes to production code
