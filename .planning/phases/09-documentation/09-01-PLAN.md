<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → sigstore/OIDC | cosign exchanges GitHub Actions OIDC token for a Fulcio signing certificate; the OIDC token is scoped to the workflow run |
| GitHub Actions runner → homebrew-docker-deploy repo | HOMEBREW_TAP_TOKEN grants write access to a separate repo; token is stored as an encrypted Actions secret |
| User machine → checksums.txt | install.sh downloads checksums.txt and verifies SHA256 before executing binary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | Tampering | checksums.txt | mitigate | cosign keyless signs checksums.txt; install.sh verifies SHA256 before executing binary (D-15) |
| T-09-01-02 | Elevation of Privilege | HOMEBREW_TAP_TOKEN | mitigate | PAT stored as encrypted GitHub Actions secret; never appears in workflow logs or .goreleaser.yaml source; use classic PAT with repo scope only (not broader org permissions) |
| T-09-01-03 | Spoofing | GoReleaser brews push | accept | GoReleaser authenticates to GitHub API with the PAT; attacker would need PAT to inject formula — secret rotation policy covers this |
| T-09-01-04 | Information Disclosure | COSIGN_EXPERIMENTAL env | accept | COSIGN_EXPERIMENTAL=1 enables keyless signing; no private key material ever stored — signing uses ephemeral keys tied to the OIDC token |
</threat_model>
