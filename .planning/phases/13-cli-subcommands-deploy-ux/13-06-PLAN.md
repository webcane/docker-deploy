Add path-aware sudo detection to Upload(): probe whether the target path requires elevation before running any sudo operations. User-writable paths bypass all SudoExec calls; only elevated paths invoke the full fallback chain.

Purpose: Deploys to ~/myproject should never prompt for a sudo password. Currently, every deploy attempts the sudo fallback chain regardless of whether the target is writable.
Output: needsSudo bool derived from a `test -w` probe at the top of Upload(), with conditional dispatch to SudoExec (needsSudo=true) or sshRun(nil) (needsSudo=false) for all remoteBase operations.

Dependency: This plan MUST run after Plan 13-04, because it calls SudoExec which is exported there.
