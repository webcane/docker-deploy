<objective>
Fix the first-deploy directory nesting bug in Upload().

Purpose: When remoteBase does not exist before the deploy, step 8 creates it with
`mkdir -p`. By the time step 9 runs `mv stagingDir remoteBase`, the target directory
already exists — so Unix mv moves stagingDir *inside* remoteBase instead of renaming
it to remoteBase. Files land at remoteBase/<staging-name>/ rather than remoteBase/.

The fix is a one-line insertion in the `else` branch (first-deploy path only):
immediately after step 8 creates remoteBase, remove it with `sudo rm -rf remoteBase`
so that the subsequent `mv stagingDir remoteBase` is a clean rename. Parent directories
created by `mkdir -p` remain untouched.

The repeat-deploy path (`existsBefore = true`) is correct and unchanged.

Output: upload.go with the first-deploy mv bug fixed and a regression test added.
</objective>
