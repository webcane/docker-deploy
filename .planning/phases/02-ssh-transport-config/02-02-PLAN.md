Step 1 — Build auth methods (D-01, D-02):

If "no" or anything else: return fmt.Errorf("host key verification rejected by user").

- Dial using a goroutine to enforce full SSH handshake timeout: launch a goroutine calling ssh.Dial("tcp", addr, clientConfig), send result on a channel. Use select with ctx.Done() and a time.After(cfg.Timeout) case to cancel if the handshake hangs. Return fmt.Errorf("SSH connection timed out after %v", cfg.Timeout) on timeout.

Step 4 — Auth failure formatting (D-03):
- If dial returns an error containing "unable to authenticate" or "no supported methods", return fmt.Errorf("SSH auth failed: ensure your key is loaded in ssh-agent or configured in ~/.ssh/config for host %s", cfg.Hostname).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network → ssh.Dial | Remote host sends public key; must be verified against known_hosts |
| user Stdin → TOFU prompt | User types "yes"/"no"; must not auto-accept on empty or ambiguous input |
| ~/.ssh/config → key loading | Local file may reference keys with passphrases; failed loads must be silently skipped |
| ~/.ssh/known_hosts → write | Appending new host entries; must use O_APPEND to avoid truncation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Spoofing | HostKeyCallback | mitigate | knownhosts.New() enforces strict verification; MITM returns KeyMismatchError hard fail |
| T-02-05 | Spoofing | InsecureIgnoreHostKey | mitigate | Never used; grep gate in verify confirms zero occurrences |
| T-02-06 | Elevation of Privilege | SSH agent socket | accept | Agent socket is user-owned; plugin inherits user's agent access — appropriate |
| T-02-07 | Denial of Service | Dial hang | mitigate | Goroutine + time.After(timeout) + context.Done() cancel; hard timeout enforced per CLAUDE.md Rule 2 |
| T-02-08 | Information Disclosure | TOFU fingerprint | accept | Fingerprint displayed to user before confirmation — this is the intended security UX |
| T-02-09 | Spoofing | TOFU auto-accept | mitigate | Empty or non-"yes" response treated as rejection; only exact "yes" proceeds |
</threat_model>
