Types to define:
- Host struct: fields User string, Hostname string, Port int
- FileConfig struct: fields Version int (yaml:"version"), Target TargetConfig (yaml:"target")
- TargetConfig struct: fields Host string (yaml:"host"), Path string (yaml:"path")
- Config struct: fields Host Host, Path string — the fully resolved runtime config

Functions to implement:
- ParseHost(rawURL string) (Host, error): parse an ssh:// URL using net/url. If rawURL is empty return zero Host and nil. If scheme is not "ssh" return error. Extract User from url.User.Username(), Hostname from url.Hostname(), Port from url.Port() (default 22 if absent). Return error if Hostname is empty after parsing.
- LoadFile(dir string) (FileConfig, error): construct filepath dir+"/deploy.yaml". If the file does not exist (os.IsNotExist) return zero FileConfig and nil. Otherwise unmarshal YAML using gopkg.in/yaml.v3 — add this dep with go get. Return error on unmarshal failure.
- Resolve(flagHost, flagPath string, file FileConfig, projectName string) (Config, error): implement the three-tier precedence per D-09 (flag > file > default). For Host: if flagHost non-empty call ParseHost(flagHost); else if file.Target.Host non-empty call ParseHost(file.Target.Host); else leave Host as zero value. For Path: if flagPath non-empty use flagPath; else if file.Target.Path non-empty use file.Target.Path; else use "/opt/"+projectName. Return the assembled Config and any parse error.

deploy.yaml schema (per D-10, forward-compatible with multi-target): use "version" and "target" as the two top-level keys. Future phases will add a "targets" key (plural) for named targets; the "target" singular key is the single-target default. Example structure:

    version: 1
    target:
      host: ssh://user@myhost.com:22
      path: /opt/myapp

Write internal/config/config_test.go testing all behavior cases listed above using the standard library testing package (no testify). Use table-driven tests for ParseHost and Resolve precedence cases.
