<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test runner → DinD daemon | RunCompose and PollHealth execute docker commands inside Container B's Docker daemon |
| Container B DinD → Docker Hub | nginx:alpine is pulled from Docker Hub during the test run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06-01 | Information Disclosure | compose test remote paths | accept | All paths use /opt/compose-test-* prefixes; ephemeral container; no sensitive data |
| T-08-06-02 | Denial of Service | nginx:alpine pull in CI | accept | nginx:alpine is a tiny image (~10MB); Docker Hub pull is fast; test timeout is 5m total (sufficient for pull + startup); CI runners have Docker pre-installed |
| T-08-06-03 | Tampering | DinD state pollution between tests | mitigate | Each test uses a distinct project name (compose-test-healthy, compose-test-unhealthy); t.Cleanup runs compose down to remove containers before next test; tests in D-07 must not share state |
| T-08-06-04 | Spoofing | healthcheck target port 9999 | accept | Port 9999 is chosen as a closed port in nginx:alpine; if nginx ever binds 9999, the test would false-positive (healthy instead of unhealthy); acceptable risk for a test image |
</threat_model>
