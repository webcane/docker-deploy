<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → Docker socket | testcontainers-go connects to /var/run/docker.sock on the ubuntu-latest runner |
| CI workflow → repository | GitHub Actions has write access to trigger builds |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Elevation of Privilege | Native Docker socket in CI | accept | ubuntu-latest runners are ephemeral and isolated per run; Docker socket access is required for testcontainers-go; risk is scoped to the CI job lifetime |
| T-08-02-02 | Tampering | Makefile test-integration target | mitigate | Target runs `./integration/...` package only — cannot accidentally run unit tests with the integration tag; the existing `test` target is unchanged so unit test isolation is preserved |
| T-08-02-03 | Denial of Service | CI job timeout | mitigate | `go test -timeout 5m` per D-17 ensures the job fails fast rather than hanging indefinitely |
</threat_model>
