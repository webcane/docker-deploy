Define a helper `buildLargeLocalDir(t *testing.T, n int) string` that creates a t.TempDir() and writes n small files named file-0.txt through file-N.txt (each with a few bytes of content). This is used to ensure the upload is large enough that context cancellation fires mid-transfer:
```go
func buildLargeLocalDir(t *testing.T, n int) string {
    t.Helper()
    dir := t.TempDir()
    for i := 0; i < n; i++ {
        name := fmt.Sprintf("file-%03d.txt", i)
        content := fmt.Sprintf("content of file %d\n", i)
        if err := os.WriteFile(filepath.Join(dir, name), []byte(content), 0644); err != nil {
            t.Fatalf("write test file %s: %v", name, err)
        }
    }
    return dir
}
```

Implement the following three test functions (using patterns from PATTERNS.md exactly):

**TestUpload_HappyPath**:
- dial as sshuser
- create localDir with t.TempDir(), write compose.yaml with content `"services: {}\n"`
- allocate `sudoPw := ""` and `warned := false`
- call `filetransfer.Upload(context.Background(), client, localDir, "/opt/testapp-upload-happy", []string{}, &sudoPw, &warned, false)`
- assert err == nil
- assert n > 0 (at least 1 file transferred)
- cleanup: sshExecHelper to `rm -rf /opt/testapp-upload-happy` in t.Cleanup

**TestUpload_AtomicCancel** (SC-5 — the critical test):
- dial as sshuser
- remoteBase = "/opt/testapp-atomic"
- pre-seed sentinel: `sshExecHelper(t, client, "mkdir -p /opt/testapp-atomic && echo original > /opt/testapp-atomic/sentinel-before-deploy.txt")`
- create localDir with buildLargeLocalDir(t, 50) — 50 files to ensure transfer takes long enough for cancel
- create `ctx, cancel := context.WithCancel(context.Background())`
- launch goroutine: `go func() { time.Sleep(50 * time.Millisecond); cancel() }()`
- call Upload with the cancellable context; ignore the error (cancellation returns error)
- assert sentinel content: `out := sshExecOutputHelper(t, client, "cat /opt/testapp-atomic/sentinel-before-deploy.txt")`; `strings.TrimSpace(out) == "original"`
- assert no staging dir remains: `out2 := sshExecOutputHelper(t, client, "ls /tmp/docker-deploy-* 2>/dev/null | wc -l")`; assert out2 trims to "0" (or use `"ls /tmp/docker-deploy-* 2>/dev/null && echo found || echo none"` and assert contains "none")
- cleanup: sshExecHelper to `rm -rf /opt/testapp-atomic` in t.Cleanup

Note on timing: 50ms may not be enough on a slow system. Use 100ms as the cancel delay for reliability. The 50 files (each ~20 bytes) may transfer quickly in a container. If tests run without exercising the cancel path (Upload completes before cancel), the sentinel assertion still passes (atomic move succeeded = sentinel was replaced; that is a false negative for the test). To force the cancel to fire mid-transfer, use a larger file count (100) with slightly larger content (1KB per file via `strings.Repeat("x", 1024)`). This ensures the 100ms cancel fires reliably before all files complete.

**TestUpload_SkipEnv** (D-04):
- dial as sshuser
- remoteBase = "/opt/testapp-skipenv"
- pre-seed .env: `sshExecHelper(t, client, "mkdir -p /opt/testapp-skipenv && echo 'SECRET=original' > /opt/testapp-skipenv/.env")`
- create localDir with t.TempDir(), write only `compose.yaml` (no .env in localDir)
- call Upload with excludes: `[]string{".env"}` (skip-env passes .env in excludes list per filetransfer/filter.go behavior)
- assert err == nil
- assert remote .env unchanged: `out := sshExecOutputHelper(t, client, "cat /opt/testapp-skipenv/.env")`; assert `strings.TrimSpace(out) == "SECRET=original"`
- cleanup: sshExecHelper to `rm -rf /opt/testapp-skipenv` in t.Cleanup

Important: sudoPw and warnedOnce must be allocated as local zero-values and addresses passed for each Upload call. Never share these across calls (each test is independent).
