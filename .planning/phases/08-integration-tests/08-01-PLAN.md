Create integration/testdata/Dockerfile.sshd. Use ubuntu:22.04 as base (per PATTERNS.md §Specific Ideas — maximum control over user setup).

The image must:

1. Install Docker Engine CE (via Docker's official apt repo: https://download.docker.com/linux/ubuntu) and OpenSSH server (openssh-server). Also install sudo, curl (for unhealthy HEALTHCHECK tests), and procps.

2. Configure sshd:
   - Create /var/run/sshd and set permissions
   - Edit /etc/ssh/sshd_config: set Port 22, PermitRootLogin yes, PubkeyAuthentication yes, PasswordAuthentication no, ChallengeResponseAuthentication no, UsePAM no
   - Host keys: generate RSA host key via `RUN ssh-keygen -t rsa -f /etc/ssh/ssh_host_rsa_key -N ''`

3. Generate four per-user RSA key pairs at build time via `RUN ssh-keygen`. Store private keys at well-known paths so the test suite can read them out. Suggested layout:
   - /etc/ssh/test_keys/root_rsa (private), /etc/ssh/test_keys/root_rsa.pub
   - /etc/ssh/test_keys/sshuser_rsa (private), /etc/ssh/test_keys/sshuser_rsa.pub
   - /etc/ssh/test_keys/nosudouser_rsa (private), /etc/ssh/test_keys/nosudouser_rsa.pub
   - /etc/ssh/test_keys/sudopassuser_rsa (private), /etc/ssh/test_keys/sudopassuser_rsa.pub
   - All generated with: `RUN ssh-keygen -t rsa -b 2048 -f /etc/ssh/test_keys/<user>_rsa -N ''`

4. Configure four users (per D-10):
   a. root — already exists. Write /etc/ssh/test_keys/root_rsa.pub to /root/.ssh/authorized_keys. Ensure /root/.ssh has mode 700, authorized_keys mode 600.
   b. sshuser — `RUN useradd -m -s /bin/bash sshuser`. Add to docker group (`usermod -aG docker sshuser`). Add `sshuser ALL=(ALL) NOPASSWD: ALL` to /etc/sudoers.d/sshuser (mode 440). Write sshuser_rsa.pub to /home/sshuser/.ssh/authorized_keys with correct ownership and modes.
   c. nosudouser — `RUN useradd -m -s /bin/bash nosudouser`. No sudo config. No docker group. Write nosudouser_rsa.pub to /home/nosudouser/.ssh/authorized_keys.
   d. sudopassuser — `RUN useradd -m -s /bin/bash sudopassuser`. Set a password (`RUN echo 'sudopassuser:testpassword' | chpasswd`). Add `sudopassuser ALL=(ALL) ALL` (with password required) to /etc/sudoers.d/sudopassuser (mode 440). Write sudopassuser_rsa.pub to /home/sudopassuser/.ssh/authorized_keys.

5. Docker daemon configuration: create /etc/docker/daemon.json with `{"storage-driver": "overlay2"}` for reliable DinD operation.

6. Entrypoint script at /entrypoint.sh:
   ```
   #!/bin/sh
   set -e
   # Start Docker daemon in background
   dockerd --host=unix:///var/run/docker.sock &
   # Wait for Docker socket
   for i in $(seq 1 30); do
     if docker info >/dev/null 2>&1; then break; fi
     sleep 1
   done
   # Start sshd in foreground
   exec /usr/sbin/sshd -D -e
   ```
   Set `chmod +x /entrypoint.sh`. Use `CMD ["/entrypoint.sh"]`.

7. EXPOSE 22

The image must be buildable without network access during tests (all apt installs happen at Docker build time).

10. dialContainer() helper — used by preflight_test.go, filetransfer_test.go, compose_test.go to get a *gossh.Client for Container B as a specific user:
    ```go
    func dialContainer(t *testing.T, user string) *gossh.Client {
        t.Helper()
        signer, ok := sshB.signers[user]
        if !ok {
            t.Fatalf("no signer for user %q", user)
        }
        khFile := seedKnownHosts(t, sshB.host, sshB.port, sshB.hostKey)
        kh, err := knownhosts.New(khFile)
        if err != nil {
            t.Fatalf("knownhosts.New: %v", err)
        }
        clientCfg := &gossh.ClientConfig{
            User:            user,
            Auth:            []gossh.AuthMethod{gossh.PublicKeys(signer)},
            HostKeyCallback: kh,
            Timeout:         15 * time.Second,
        }
        client, err := gossh.Dial("tcp", fmt.Sprintf("%s:%d", sshB.host, sshB.port), clientCfg)
        if err != nil {
            t.Fatalf("dial container as %q: %v", user, err)
        }
        t.Cleanup(func() { client.Close() })
        return client
    }
    ```
    This uses gossh.Dial() directly (not internalssh.Dial()) to bypass the SSH agent chain (per PATTERNS.md note 2). HostKeyCallback uses knownhosts.New() — never InsecureIgnoreHostKey (CLAUDE.md Rule 1).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test runner → SSH container | Test code connects to container SSH port; containers are ephemeral and isolated |
| Dockerfile.sshd build → Docker daemon | Image is built locally or in CI; no external registry pull for custom image |
| Container SSH keys → test code | Private keys are read from container via testcontainers exec API, never written to disk permanently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01-01 | Spoofing | dialContainer() HostKeyCallback | mitigate | Use knownhosts.New() with captured host key — never InsecureIgnoreHostKey (CLAUDE.md Rule 1 enforced by grep gate in acceptance_criteria) |
| T-08-01-02 | Elevation of Privilege | Dockerfile.sshd PermitRootLogin | accept | Root login is required for CHECK-07 test coverage; container is ephemeral and isolated to test network; no production traffic |
| T-08-01-03 | Elevation of Privilege | DinD privileged container | accept | Privileged mode required for Docker-in-Docker; testcontainers isolates to test run; risk is CI runner compromise, not data exfiltration |

| T-08-01-05 | Denial of Service | Container startup timeout | mitigate | WaitingFor: ForListeningPort with 90s (Container A) and 120s (Container B) startup timeouts; TestMain log.Fatal on startup failure prevents hung CI |
