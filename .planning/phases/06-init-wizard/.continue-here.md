### From "Root credential input" discussion:

The wizard uses `huh` for all prompts. Question sequence:

1. **Host prompt** — accepts `sshuser@192.168.1.99`, `sshuser@minipc`, or bare hostname/IP
   - If bare hostname/IP (no `user@` prefix) → ask for username separately
2. **SSH key resolution** — if hostname matches entry in `~/.ssh/config` → use its `IdentityFile`
   - Otherwise → present list of keys from `~/.ssh/` for user to select
3. **Compose file** — only ask if no `compose.yaml` or `docker-compose.yaml` found in cwd
4. **Project name** — default: cwd basename (or name from compose label / DOCKER label if present)
5. **Target directory** — default: `/opt/<project>`; user can set e.g. `~/<project>`
6. **Deploy mode** — ask: copy files only, or copy + run `docker compose up`?
7. **Exclusions/inclusions** — NOT asked in wizard (deferred / handled by deploy.yaml)
8. **On connect:** if SSH key not in agent → prompt for password
9. **If sudo required and passwordless sudo not configured** → prompt for sudo password
