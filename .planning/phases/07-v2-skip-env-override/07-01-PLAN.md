<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deploy.yaml → Resolve() | User-controlled file; skip_env: true can suppress .env upload silently |
| --skip-env flag → runDeploy() | Operator-supplied flag; bypasses .env upload without interactive confirmation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Information Disclosure | skip_env / .env exclude | mitigate | Per D-09: print `WARNING: .env not uploaded — remote .env left unchanged` to stderr whenever SkipEnv is true, ensuring operator awareness; this warning is part of the rollup system (Plan 02) so it surfaces even without --verbose |
| T-07-01-02 | Tampering | defaultExcludes expansion | accept | New entries are dev-tooling directories only; no production artifact is inadvertently excluded; tool's core value proposition (.env always uploaded by default) is preserved |
| T-07-01-03 | Information Disclosure | FlagOpts struct | accept | Struct holds flag values in process memory only; no persistence; no new attack surface beyond existing positional params |
</threat_model>
