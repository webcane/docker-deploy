<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → --skip-env flag | Operator intentionally suppresses .env upload; remote secrets silently diverge from local |
| operator → --verbose flag | Detailed output (file names, SSH commands) written to stderr; if stderr is logged, file paths are exposed |
| warning rollup → operator | Non-verbose mode hides individual warnings; operator may miss actionable issues |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Information Disclosure | --skip-env | mitigate | Per D-09: always print `WARNING: .env not uploaded — remote .env left unchanged` to stderr; without --verbose this becomes the rollup message so it is never silently swallowed — the rollup fires whenever warnings > 0 |
| T-07-02-02 | Repudiation | warning rollup | mitigate | The rollup message `WARN: there are some warnings during deployment. For more details use --verbose flag` is always printed to stderr when any non-blocking warning occurred; operator cannot claim ignorance of warnings |
| T-07-02-03 | Information Disclosure | --verbose stderr output | accept | Per-file paths and SSH commands are written to stderr which is operator-controlled; logging pipelines that capture stderr receive file names — acceptable for a deploy tool used by the operator themselves |
| T-07-02-04 | Elevation of Privilege | Upload() verbose param | accept | verbose param is a read-only behavior switch; it does not change SSH commands, paths, or auth logic; no privilege escalation risk |
</threat_model>
