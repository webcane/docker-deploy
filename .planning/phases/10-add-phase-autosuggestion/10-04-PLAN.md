- `contrib/install-completions.sh`: a portable POSIX shell script that downloads the matching completion file from the latest release tarball and installs it to the correct fpath location for the detected shell

  <name>Task 3: Create contrib/install-completions.sh for non-Homebrew users</name>
  <files>contrib/install-completions.sh</files>
  <read_first>

    - .planning/notes/completion-rework-design.md (manual install fallback section)
  </read_first>
  <action>
    Create `contrib/install-completions.sh` as an executable POSIX `/bin/sh` shell script (chmod 0755) with the following behaviour:

    1) Shebang: `#!/bin/sh` (POSIX, not `#!/bin/bash`).
    2) `set -eu` near the top for safety. Do NOT use `set -o pipefail` (not POSIX).
    3) Constants:
       - `REPO="webcane/docker-deploy"`
       - `VERSION` — accept override via the env var `INSTALL_VERSION`; default to `latest` (resolved by downloading the latest release tarball, mirroring `install.sh`).
    4) Auto-detect shell from `$SHELL` (basename). Branch:
       - `zsh`: download only `_docker-deploy`.
       - `bash`: download only `docker-deploy.bash`.
       - anything else: print an error to stderr — "unsupported shell: $shell (supported: bash, zsh)" — and exit 1.
    5) Determine install destination:
       - For zsh:
         - If `/opt/homebrew/share/zsh/site-functions/` is a directory (Homebrew Apple Silicon), install there.
         - Else if `/usr/local/share/zsh/site-functions/` is a directory (Homebrew Intel), install there.
         - Else fall back to `${HOME}/.zsh/completions/`, creating it with `mkdir -p`.
       - For bash:
         - If `/opt/homebrew/share/bash-completion/completions/` is a directory, install there.
         - Else if `/usr/local/share/bash-completion/completions/` is a directory, install there.
         - Else fall back to `${HOME}/.bash_completion.d/`, creating it with `mkdir -p`.
    6) Download method: use `curl -fsSL` to fetch the release tarball
       `https://github.com/${REPO}/releases/download/${VERSION}/docker-deploy_$(uname -s | tr '[:upper:]' '[:lower:]')_$(uname -m).tar.gz` if the tarball ships completions (per 10-05 / goreleaser extra_files), OR fetch the raw completion file directly from `https://raw.githubusercontent.com/${REPO}/${VERSION}/contrib/{filename}`. Use the raw-content approach (simpler, no tarball extraction). If `VERSION` is `latest`, resolve via the GitHub redirect on `releases/latest` or simply use the `master` ref as a fallback documented in a comment.
    7) After writing the file, print a success message to stdout indicating where it was installed AND any follow-up shell config the user should add (for `~/.zsh/completions/` add `fpath=(~/.zsh/completions $fpath)` to `~/.zshrc`; for `~/.bash_completion.d/` add a one-liner that sources the file from `~/.bashrc`).
    8) Make the file executable: `chmod +x contrib/install-completions.sh` after creation.

    Keep the script under ~120 lines. Do NOT inline both shells' completion content — only download the relevant one.

    Sanity-check the script with `sh -n contrib/install-completions.sh` (POSIX syntax check) before considering the task done.
  </action>
