<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell → completion func | Tab-press triggers cobra to call the registered completion function; no network I/O |
| filesystem → completion func | Reads user-owned deploy.yaml and ~/.ssh/config at completion time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Tampering | Shell injection via --host completion candidate | accept | Completion candidates are printed to stdout as text; they are not executed; cobra handles the display protocol |
| T-10-02-02 | Tampering | Path traversal via --compose-file completion | accept | os.Stat is called only on the two hardcoded filenames "compose.yaml" and "docker-compose.yml"; no user input reaches os.Stat |
| T-10-02-03 | Spoofing | Malicious ~/.ssh/config causing panic in ListHosts | mitigate | ListHosts uses bufio.Scanner; returns nil on any error per D-03; no dynamic execution |
| T-10-02-04 | Denial of Service | completion subcommand accepts only ValidArgs{"bash","zsh"} — fish/powershell rejected by cobra | mitigate | cobra.ExactValidArgs(1) + ValidArgs: []string{"bash","zsh"} rejects unsupported shells before RunE |
| T-10-02-05 | Information Disclosure | deploy.yaml host value exposed in shell completion candidates | accept | File is user-owned; completion candidates are shown only to the user who already has access to deploy.yaml |
</threat_model>
