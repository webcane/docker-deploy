    Export: func HostCompletionFunc(_ *cobra.Command, _ []string, _ string) ([]cobra.Completion, cobra.ShellCompDirective)
      1. Get cwd via os.Getwd(); on error return nil, cobra.ShellCompDirectiveNoFileComp (D-03)
      2. Try config.LoadFile(cwd); if err == nil and fc.Target.Host != "", append fc.Target.Host to hosts
      3. Get home via os.UserHomeDir(); if err == nil, call sshconfig.ListHosts(filepath.Join(home, ".ssh", "config")) and append all results to hosts
      4. Deduplicate hosts using dedupStrings (preserve order, first occurrence wins)
      5. Return (hosts as []cobra.Completion), cobra.ShellCompDirectiveNoFileComp

    Export: func PathCompletionFunc(_ *cobra.Command, _ []string, _ string) ([]cobra.Completion, cobra.ShellCompDirective)
      1. Get cwd via os.Getwd(); on error return nil, cobra.ShellCompDirectiveNoFileComp (D-03)
      2. Return []cobra.Completion{"/opt/" + filepath.Base(cwd)}, cobra.ShellCompDirectiveNoFileComp (D-06)

    Export: func ComposeFileCompletionFunc(_ *cobra.Command, _ []string, _ string) ([]cobra.Completion, cobra.ShellCompDirective)
      1. Iterate []string{"compose.yaml", "docker-compose.yml"}
      2. For each name, call os.Stat(name); if err == nil append cobra.Completion(name) to suggestions
      3. Return suggestions, cobra.ShellCompDirectiveNoFileComp (D-07)
      (Empty suggestions slice is valid — cobra treats nil and []cobra.Completion{} identically)

    Unexported helper: func dedupStrings(in []string) []cobra.Completion
      Uses a seen map[string]bool to deduplicate while preserving first-occurrence order.
      Returns []cobra.Completion (since cobra.Completion = string, cast each element).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell -> completion func | Tab-press triggers cobra to call the registered completion function; no network I/O |
| filesystem -> completion func | Reads user-owned deploy.yaml and ~/.ssh/config at completion time; both files are user-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Tampering | Shell injection via --host completion candidate from deploy.yaml | accept | Completion candidates are printed to stdout as text; they are not executed by the shell; cobra handles the display protocol |
| T-10-02-02 | Tampering | Path traversal via --compose-file completion | accept | os.Stat is called only on the two hardcoded filenames "compose.yaml" and "docker-compose.yml" in ComposeFileCompletionFunc; no user input reaches os.Stat |
| T-10-02-03 | Spoofing | Malicious ~/.ssh/config content causing panic in ListHosts | mitigate | ListHosts uses bufio.Scanner; returns nil on any error per D-03; HostCompletionFunc wraps the call with silent error handling |
| T-10-02-04 | Denial of Service | completion subcommand accepts only ValidArgs{"bash","zsh"} | mitigate | cobra.ExactValidArgs(1) + ValidArgs: []string{"bash","zsh"} rejects unsupported shells before RunE fires |
| T-10-02-05 | Information Disclosure | deploy.yaml host value exposed in shell completion candidates | accept | File is user-owned; completion candidates are shown only to the user who already has read access to deploy.yaml |
</threat_model>
