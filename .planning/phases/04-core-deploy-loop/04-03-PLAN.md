<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| --compose-file CLI flag → RunCompose | User-controlled filename enters the remote SSH command |
| resolved.ComposeFile → ssh exec | The basename is concatenated (not quoted) into the compose -f argument |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Injection | --compose-file value in remote command | mitigate | filepath.Base(resolved.ComposeFile) == resolved.ComposeFile validation in runDeploy() rejects any value containing path separators (slashes, backslashes); an attacker cannot escape the remote_path/ prefix via the compose-file argument |
| T-04-03-02 | Elevation of Privilege | compose.RunCompose() call with client open | accept | client is already authenticated by Dial(); RunCompose opens no new connections; session.Start() runs a fully-formed command, not a shell; no additional privilege surface introduced |
| T-04-03-03 | Denial of Service | Blocking on session.Wait() for long-running compose | accept | compose up -d returns quickly (detached); long-running containers do not block the session after detach. Compose startup output is finite |
| T-04-03-04 | Information Disclosure | "Deploy failed: docker compose exited with code N" on stderr | accept | Exit code is an integer; no secrets or filesystem paths exposed in failure message |
</threat_model>
