<objective>
Close two UAT gaps from Phase 15:

1. [major] YAML healthcheck field typos are silently ignored. `yaml.Unmarshal` in `LoadFile()` drops unknown fields — a user who types `retrise: 3` gets `retries=0` with no feedback. Fix: use `yaml.NewDecoder` with `KnownFields(true)` so any unknown field is a parse error.

2. [minor] `--dry-run` output omits resolved healthcheck config. Users cannot verify flag/file precedence without a live deploy. Fix: add a `Healthcheck:` row to `runDryRun()` output, always visible (not gated on `--verbose`), showing `interval=Xs timeout=Ys retries=N` or `disabled`.

Purpose: Users who misconfigure healthcheck YAML currently get silent wrong behaviour; `--dry-run` cannot be used to verify config precedence.
Output: Strict YAML parsing in `LoadFile()`, new tests covering both strict-mode scenarios, and healthcheck row in dry-run summary.
</objective>
