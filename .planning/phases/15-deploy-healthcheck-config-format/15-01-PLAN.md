<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flags / deploy.yaml → config.Resolve() | Untrusted user input (strings, integers) crosses into config parsing |
| config values → health.PollHealth() | Resolved durations flow into time.NewTicker / time.NewTimer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01-01 | Tampering / DoS | Resolve() duration parsing | mitigate | Use time.ParseDuration which rejects malformed strings; return explicit error before any timer is created — no panic path |
| T-15-01-02 | DoS | Negative retries / negative durations | mitigate | Validate `Retries >= 0`; rely on time.ParseDuration to reject negatives in strings (Go does accept negative durations like "-5s", so explicitly reject Interval/Timeout < 0 after parsing) |
| T-15-01-03 | Information disclosure | Error messages echo invalid input | accept | Reflecting the user's own input back is standard CLI behaviour and not sensitive — no PII or secrets involved |
| T-15-01-04 | Injection | Duration strings flow into shell commands | accept | Duration strings never reach the shell; they only feed `time.NewTicker` / `time.NewTimer`. Path/host args already use ShellQuote elsewhere |
</threat_model>
