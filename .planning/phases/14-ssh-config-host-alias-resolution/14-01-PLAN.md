  Add exported LookupHost(configPath, alias string) (HostEntry, bool) function. The function:
  - Opens configPath; returns HostEntry{}, false on open error (file absent is not an error to caller)
  - Scans line by line using bufio.Scanner; skip blank lines and "#" comments
  - Tracks active bool and a found bool; when a "host" keyword line is reached, set active = hostMatches(pattern, alias) for any of the space-separated patterns; reset hostnameSeen bool for new block

  - When a second "host" keyword is reached and active was true and found is false, set found = true (the block ended)
  - After scan loop: if active is still true (last block in file), set found = true
  - Per D-07: after scanning, if entry.HostName == "" set entry.HostName = alias
  - Per D-08: Port 0 means caller defaults to 22 — do not set a default here
  - Per D-09: User "" means caller defaults to system user — do not set a default here
  - Per D-11: "include" keyword line → continue (skip silently, no error); add TODO comment: "TODO: Include directives not implemented — only the named config file is parsed"
  - Return entry, found

  - If found && len(entry.IdentityFiles) > 0, use entry.IdentityFiles; else fall through to defaultIdentityFiles()

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ~/.ssh/config → sshconfig parser | File is user-controlled; malformed or hostile content must not crash the process |
| alias string → SSH dial target | Alias resolves to HostName used as known_hosts key; must use resolved HostName not alias label |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01-01 | Spoofing | LookupHost / DialConfig.Hostname | mitigate | Per D-12: DialConfig.Hostname = entry.HostName (the real hostname from ssh config), not the alias label — known_hosts verification uses the real hostname so a spoofed alias cannot bypass host key checking |
| T-14-01-02 | Tampering | ~/.ssh/config parser (Include) | accept | Include directives are silently skipped (D-11); an attacker who can write ~/.ssh/config can already control ssh connections directly — skipping Include limits blast radius |
| T-14-01-03 | Information Disclosure | Error message (alias not found) | accept | Error names the config file path and alias; both are already known to the operator; no secret data exposed |
| T-14-01-04 | Elevation of Privilege | InsecureIgnoreHostKey | mitigate | No InsecureIgnoreHostKey() call anywhere; grep gate in verify step enforces this; alias resolution only affects HostName lookup, not knownhosts verification |
</threat_model>
