## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deploy.yaml → config.Resolve() | File contents are parsed and validated; invalid values return errors rather than being accepted |
| validate subcommand → filesystem | Read-only: only reads deploy.yaml from cwd; no SSH, no writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-03-01 | Information Disclosure | validate error output to stderr | accept | Error messages describe config parse failures; no credentials in deploy.yaml are echoed back |
| T-13-03-02 | Spoofing | SSH not called by validate | accept | validate explicitly does not dial SSH (D-08); no risk of connecting to wrong host during validation |
| T-13-03-03 | Elevation of Privilege | runValidate reads from cwd only | accept | Read-only filesystem access; os.Stat + config.LoadFile(cwd) do not follow symlinks to elevated paths |
</threat_model>
