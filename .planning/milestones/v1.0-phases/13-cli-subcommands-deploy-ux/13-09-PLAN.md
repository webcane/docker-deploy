<objective>
Fix the double file-listing in verbose mode: local filenames appear twice — once as `  -> filename` arrows during SFTP staging upload, and again as `Local files (N):` in the pre-confirm diff block.

The root cause is that the per-file `  -> %s` line at upload.go line 348 fires unconditionally during the SFTP upload loop (step 6), which always runs before the `existsBefore` check and the diff block. Both outputs show the same local file list.

The fix replaces the per-file arrows with a single `Uploading N files...` summary line printed BEFORE the loop when verbose is true. The diff block's `Local files (N):` section continues to show individual filenames before the confirm prompt — that is the appropriate place for the detailed list.

Purpose: Eliminate redundant output in verbose mode so operators see each filename exactly once.
Output: Updated upload loop in upload.go; updated TestUploadVerbose_PerFileStderr to expect the new summary line.
</objective>
