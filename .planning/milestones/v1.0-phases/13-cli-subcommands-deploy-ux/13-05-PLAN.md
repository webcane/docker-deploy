<objective>
Add verbose pre-confirm file diff to Upload(): display local and remote file lists to stderr before the "Replace all contents?" prompt when --verbose is active.

Purpose: Operators deploying to a live server need to see what will change before confirming a destructive replace. The diff shows both sides so they can spot unexpected files.
Output: Verbose diff block inside Upload() that prints local (from WalkFiles) and remote (from SFTP ReadDir) file lists, truncated at 20, before the confirm prompt.

Architecture note (RESEARCH.md Pitfall 7): The confirm prompt currently lives in main.go (lines 295-328) and fires before Upload() is called. The SFTP client only opens inside Upload(). To show the diff before the prompt while keeping both in sequence, move the confirm prompt logic inside Upload(). Upload() already receives force bool in the config path; pass it explicitly if not already present. The existence check session in main.go (lines 294-327) is replaced by Upload()'s internal existsBefore + prompt logic.

Dependency: This plan runs AFTER Plan 13-04. The Upload() signature it receives already has `creds *SudoCreds` in place of `sudoPw *string`. The `force bool` parameter is added on top of that completed signature.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote file listing → stderr | SFTP ReadDir returns filenames only; no file contents are printed |
| stdin confirm prompt | User input is read from os.Stdin; only "y"/"yes" proceeds — default is no-op |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05-01 | Information Disclosure | Remote file names printed to stderr | accept | File names are metadata the operator is expected to see; no file contents or credentials are printed |
| T-13-05-02 | Tampering | SFTP ReadDir failure silently falls back | accept | ReadDir failure shows "(unable to list: err)" — operator informed but deploy proceeds; partial listing cannot be confused for a complete one |
| T-13-05-03 | Denial of Service | ReadDir on very large remote dirs | mitigate | Truncation at 20 filenames (D-18) prevents excessive stderr output; ReadDir returns all entries but only 20 are printed |
</threat_model>
