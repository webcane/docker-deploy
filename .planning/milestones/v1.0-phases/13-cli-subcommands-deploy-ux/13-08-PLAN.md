<objective>
Fix `docker deploy version` showing the `Built:` line in dev/untagged builds.

The Makefile always injects `buildTime` via ldflags (`-X main.buildTime=$(shell date ...)`), so the existing guard `if buildTime != "unknown"` is always true for `make build` regardless of whether this is a tagged release or a local dev build. The correct discriminator is `version != "dev"`: `make build` sets `version` to `dev` for untagged commits; GoReleaser injects the semver tag on tagged releases.

Purpose: Uphold the spec (D-03) that dev builds show 3 lines (Version, Git commit, OS/Arch) and tagged builds show 4 lines (+ Built).
Output: Updated condition in `runVersionTo()` and an additional test asserting that a dev build with an injected `buildTime` still omits the Built line.
</objective>
