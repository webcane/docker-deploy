  <action>
    Run `make lint-fix` first to auto-fix any auto-fixable issues. Then run `make lint` to see remaining findings.

    For each remaining finding, read the flagged file and fix the issue inline. Common patterns:

    **gosec findings:** G104 (unchecked errors) — add explicit `if err != nil` check or use `_` assignment with a comment if the error is genuinely ignorable. G304 (file path from variable) — wrap with filepath.Clean() if needed. Do not use `//nolint:gosec` except as a last resort with an inline comment explaining why the risk is accepted.

    **wrapcheck findings:** Errors returned from external package calls must be wrapped with `fmt.Errorf("...: %w", err)` to add context. Exception: errors already wrapped by a prior call in the same function.

    **errorlint findings:** `errors.Is` / `errors.As` instead of direct `==` comparison on error values. Fix by replacing `err == someErr` with `errors.Is(err, someErr)`.

    **gocognit / nestif findings:** Reduce complexity by extracting helper functions or flattening early-return patterns. Do not suppress with `//nolint` directives.

    **bodyclose findings:** HTTP response bodies not closed — add `defer resp.Body.Close()`.

    **noctx findings:** HTTP requests without context — pass `context.Background()` or a real context to `http.NewRequestWithContext`.

    After all fixes, run `make lint` and confirm it exits 0 with no output.

    If any finding genuinely cannot be fixed (e.g. a gosec finding that is a documented false positive for this codebase), add a targeted `//nolint:gosec // reason: <explanation>` inline comment on the specific line only — never file-level or package-level suppress.
  </action>
