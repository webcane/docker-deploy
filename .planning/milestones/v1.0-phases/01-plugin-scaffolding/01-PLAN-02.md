<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → release artifacts | GoReleaser produces binaries on GitHub-managed runners and uploads to GitHub Releases |
| Downloader → release archive | End users download archives from GitHub Releases |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Release binary download | mitigate | `checksums.txt` (SHA256) is generated by GoReleaser and attached to every release. Users must verify checksum before install. |
| T-02-02 | Tampering | GitHub Actions workflow file | accept | Workflow files live in the repo; any modification is visible in git history and requires push access to the repository. |
| T-02-03 | Elevation of Privilege | `GITHUB_TOKEN` in release workflow | mitigate | `permissions: contents: write` scoped to the release job only — no read of secrets, no other repo permissions granted. |
| T-02-04 | Spoofing | GoReleaser action version pinning | accept | Using `goreleaser/goreleaser-action@v6` major-version tag. For Phase 1 skeleton this is acceptable; pin to a full SHA in a future security-hardening phase if required. |
</threat_model>
