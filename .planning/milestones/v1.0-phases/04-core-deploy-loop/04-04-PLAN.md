<objective>
Implement a structured auth fallback sequence for file copy during deploy execution, enabling deployment to succeed via direct copy, passwordless sudo, or interactive sudo password prompt without blocking at preflight.

**Why this matters:** Preflight CHECK-05 (passwordless sudo verification) was reclassified as a warning, not a blocker. This plan implements the actual fallback logic during deploy execution so that valid deployments (root user, user with sudo password) can proceed instead of failing.

**Auth sequence:**
1. Try direct copy (no privilege escalation)
2. Try copy with passwordless sudo (sudo without prompt)
3. Prompt user for sudo password interactively with warning that passwordless sudo isn't configured
4. Fail only when all paths exhausted (bad/no password, timeout)

**Output:** Enhanced Upload() function in filetransfer package with auth fallback logic, interactive password prompts via golang.org/x/term, comprehensive test coverage for all three auth paths, integration into runDeploy().
</objective>

### Task 2: TDD — Implement Auth Fallback Core Function

<action>
Implement tryAuthFallback(sftpClient, client, localPath, remotePath, targetUser, sudoPw *string) (success bool, finalErr error) in upload.go:

1. **Direct copy attempt:** Call sftpClient.Copy(localPath, remotePath) directly. If succeeds, return (true, nil). If fails due to permission denied, continue to next step.
2. **Passwordless sudo attempt:** If direct copy failed, try `sudo mv staging-to-target` using an SSH session with stdin closed (no password prompt). If succeeds, return (true, nil). If fails, continue to next step.
3. **Interactive password prompt:** Prompt user with "Enter sudo password for <targetUser>:" using golang.org/x/term.ReadPassword(). Attempt sudo with provided password up to 3 times. If any attempt succeeds, return (true, nil). If all 3 attempts fail or timeout, return (false, error describing exhausted paths).
4. **Root user fast path:** Before steps 1-3, check if targetUser == "root". If true, skip all sudo attempts and use direct copy only (no privilege escalation). Emit warning: "deploying as root — this is dangerous".

Warning messages for each path:
- Direct copy failure: (silent, try next path)
- Passwordless sudo failure: "passwordless sudo not configured for <targetUser>; attempting interactive password prompt..."
- Interactive password attempt N/3: (silent, try next iteration)
- All paths exhausted: "ERROR: could not write to target directory — no valid auth path available (tried direct copy, passwordless sudo, interactive password)"
</action>

</read_first>
