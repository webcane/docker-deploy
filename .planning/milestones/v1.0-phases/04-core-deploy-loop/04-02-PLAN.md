    1. Construct remote command string: "docker compose -f " + filetransfer.ShellQuote(remotePath) + "/" + composeFile + " up -d --remove-orphans" (per D-04, D-05, D-11). Note: composeFile is a basename with no slashes — do not wrap in ShellQuote.

    2. Open new session: session, err := client.NewSession(); defer session.Close().

    3. TTY detection and output routing (per D-01, D-02, D-03):
       - isTTY := term.IsTerminal(int(os.Stdout.Fd()))
       - If isTTY:
         - Get terminal dimensions via term.GetSize(int(os.Stdout.Fd())); use w=80, h=24 as fallback if error.
         - Call session.RequestPty("xterm-256color", h, w, gossh.TerminalModes{gossh.ECHO: 0, gossh.TTY_OP_ISPEED: 14400, gossh.TTY_OP_OSPEED: 14400}).
         - session.Stdout = os.Stdout (PTY merges stdout+stderr — both go to os.Stdout per D-01).
         - session.Stderr = os.Stdout (intentional PTY behavior per D-01 comment).
       - If not TTY:
         - stdoutPipe, _ := session.StdoutPipe(); stderrPipe, _ := session.StderrPipe().
         - Launch two goroutines: io.Copy(os.Stdout, stdoutPipe) and io.Copy(os.Stderr, stderrPipe).
         - Use sync.WaitGroup to wait for both goroutines after session.Wait().

    4. Call session.Start(cmd) — not session.Run() (per discretion note in CONTEXT.md). Return fmt.Errorf("starting compose session: %w", err) on error.

    5. If not TTY: wait for both goroutine drains (wg.Wait()) then call session.Wait(). If TTY: call session.Wait() directly (PTY drains inline).

    6. Exit code propagation (per D-12, DEPLOY-05):
       - If waitErr != nil:
         - var exitErr *gossh.ExitError
         - if errors.As(waitErr, &exitErr): code := exitErr.ExitStatus(); fmt.Fprintf(os.Stderr, "Deploy failed: docker compose exited with code %d\n", code); return fmt.Errorf("docker compose exited with code %d", code).
         - else: return fmt.Errorf("compose session wait: %w", waitErr).
       - Return nil on success.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remotePath → SSH exec command | remotePath comes from Config.Path, which originates from --path flag or deploy.yaml |
| composeFile → SSH exec command | composeFile comes from Config.ComposeFile, which originates from --compose-file flag, deploy.yaml, or auto-detect |
| PTY allocation → remote shell | RequestPty sets terminal type string; could influence remote $TERM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering / Injection | remotePath in compose exec command | mitigate | filetransfer.ShellQuote() wraps remotePath in single quotes with embedded-quote escaping; prevents shell injection from malicious paths |
| T-04-02-02 | Tampering / Injection | composeFile in compose exec command | mitigate | Plan 01 and Plan 03 validate filepath.Base(composeFile) == composeFile before passing to RunCompose; no slashes allowed; no ShellQuote needed for a validated basename |
| T-04-02-03 | Elevation of Privilege | PTY allocation via RequestPty | mitigate | No shell escape vector: RunCompose runs a fully-constructed command via session.Start(), not a shell. PTY provides a terminal environment for compose rendering, not interactive input. session.Stdin is not connected to os.Stdin. |
| T-04-02-04 | Denial of Service | Goroutine leak on non-TTY pipe drain | mitigate | sync.WaitGroup ensures both io.Copy goroutines complete before RunCompose returns; no goroutine leak |
| T-04-02-05 | Spoofing | ExitError type assertion on session.Wait() | accept | errors.As(waitErr, &exitErr) is the idiomatic gossh pattern; non-ExitError errors are wrapped and returned, not silently swallowed |
</threat_model>
