  <action>
    Package declaration: `package health`

    Define PollHealth signature:
    `func PollHealth(ctx context.Context, client *gossh.Client, projectName string, cfg config.Config) error`

    Implementation steps:

    Step 1 — Enumerate containers: run `docker ps --filter label=com.docker.compose.project=` + ShellQuote(projectName) + ` --format '{{.Names}}'` via session.Output(). Parse output: split by newline, trim whitespace, filter empty strings. If no containers returned → return nil (nothing to poll).

    Step 2 — Poll loop: use a ticker with time.Duration(cfg.HealthInterval) * time.Second and a timeout timer with time.Duration(cfg.HealthTimeout) * time.Second. Loop:
    - On ticker tick: for each container, run `docker inspect --format '{{.State.Health.Status}}' ` + ShellQuote(container) via session.Output() in a new session. If session.Output() returns error (e.g. container exited) treat as "unknown" — print warning, continue.
    - Health status handling per container:
      - "healthy" → mark container as done
      - "unhealthy" → fmt.Fprintf(os.Stderr, ...), return fmt.Errorf("health: container %s is unhealthy", container) immediately
      - "" or "none" → fmt.Fprintf(os.Stderr, "Warning: container %s has no HEALTHCHECK defined\n", container); mark as done (no-healthcheck is not polled further — treat as pass on first observation)
      - "starting" → continue polling
    - If all containers are done (healthy or no-healthcheck) → fmt.Fprintf(os.Stdout, "Health check passed: all containers healthy\n"), return nil
    - On timeout timer fire → fmt.Fprintf(os.Stderr, "Health check timed out after %ds: container %s is still starting\n", cfg.HealthTimeout, containerForFirstStillStarting), return fmt.Errorf("health: timed out waiting for containers to become healthy")
    - On ctx.Done() → return fmt.Errorf("health: context cancelled: %w", ctx.Err())

    Each docker ps and each docker inspect runs in a separate client.NewSession() call. Close each session after Output() returns.

    For tests: same approach as preflight — define a narrow sessionRunner interface or use a fake that captures commands and returns configurable output. The poll loop's time.Sleep / ticker can be made fast in tests by injecting a small HealthInterval (e.g. Config with HealthInterval=0 treated as 1, or tests pass a Config with very short intervals and the ticker naturally expires). Use a fake time source or accept that tests use real time.Sleep with very short intervals (1ms). Document the approach in test file comments.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSH exec output → health status parsing | Remote host controls docker inspect stdout; adversarial output could influence status determination |
| projectName → remote filter label | projectName comes from filepath.Base(cwd) — operator-controlled directory name injected into docker ps filter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Injection | docker ps filter — projectName injected into shell command | mitigate | ShellQuote(projectName) wraps the value; established pattern from Phase 3 |
| T-05-03-02 | Injection | docker inspect — container name from docker ps output used in next command | mitigate | Container names returned by docker ps are Docker-managed identifiers; ShellQuote() applied before use in inspect command |
| T-05-03-03 | Spoofing | Health status string from docker inspect | accept | Status parsing uses exact string matching ("healthy", "unhealthy", "starting", "", "none"); unexpected values fall through to "starting" poll continuation — safe default |
| T-05-03-04 | DoS | Infinite poll loop if timeout logic is wrong | mitigate | timeout timer is a time.After(HealthTimeout * second) select branch that always fires; test covers timeout path |
</threat_model>
