  <action>
    Package declaration: `package preflight`

    Define CheckResult struct with fields Name string, Status string (values: "pass", "warn", "fail"), Message string. This is the structured result type for Phase 7 verbose extension (per CONTEXT.md specifics section).

    Define RunPreflightChecks signature:
    `func RunPreflightChecks(ctx context.Context, client *gossh.Client, cfg config.Config) ([]CheckResult, error)`

    Implement checks in order. Fail-fast on first hard-block error (D-04):

    checkDockerInstalled (CHECK-01): run `docker --version` via session.Output(). Non-zero exit → return error `fmt.Errorf("preflight: docker not installed on remote host")`. No output needed on pass.

    checkComposeV2 (CHECK-02): run `docker compose version` via session.Output(). On success (exit 0) → pass. On failure: run second session with `docker-compose --version`. If that also fails → error `fmt.Errorf("preflight: docker compose v2 is not installed (docker compose plugin required; docker-compose v1 is EOL since June 2023)")`. If v1 found → same error message (v1 is EOL, hard block per D-05).

    checkDaemon (CHECK-03): run `docker info` via session.Output(). On failure → fmt.Fprintf(os.Stderr, "Warning: Docker daemon is not running on %s — compose will fail if daemon is still down at deploy time\n", cfg.Host.Hostname). Return nil (warning only, D-05, D-06).

    checkDockerGroup (CHECK-04): run `id -nG` via session.Output(). Parse output — if "docker" found in whitespace-split tokens → pass (user already in group). If not found: call checkSudo() (CHECK-05) first; if sudo fails → return its error. Then run `sudo usermod -aG docker ` + ShellQuote(cfg.Host.User) via session.Run(). On failure → fmt.Errorf("preflight: user %s is not in the docker group; fix: sudo usermod -aG docker %s (run as root or user with NOPASSWD sudo)", cfg.Host.User, cfg.Host.User).

    checkSudo (CHECK-05): run `sudo -n true` via session.Run(). On non-zero exit → fmt.Errorf("preflight: passwordless sudo required but not configured for user %s; fix: add '%s ALL=(ALL) NOPASSWD: ALL' to /etc/sudoers.d/%s on the remote host", cfg.Host.User, cfg.Host.User, cfg.Host.User). Returns nil on success.

    checkTargetDir (CHECK-06): run `test -w ` + ShellQuote(cfg.Path) via session.Run(). On success (exit 0) → pass. On failure: run `mkdir -p ` + ShellQuote(cfg.Path) via session.Run(). If mkdir succeeds → pass. If mkdir fails: call checkSudo(); if sudo fails → return its error. Run `sudo mkdir -p ` + ShellQuote(cfg.Path) and then `sudo chown ` + ShellQuote(cfg.Host.User) + ` ` + ShellQuote(cfg.Path) as two separate sessions. If either fails → fmt.Errorf("preflight: target directory %s is not writable and could not be created; fix: sudo mkdir -p %s && sudo chown %s %s", cfg.Path, cfg.Path, cfg.Host.User, cfg.Path).

    checkRootUser (CHECK-07): if cfg.Host.User == "root" → fmt.Fprintf(os.Stderr, "Warning: deploying as root is not recommended — consider creating a dedicated deploy user\n"). Return nil (never blocks, D-05).

    Execution order in RunPreflightChecks: checkDockerInstalled → checkComposeV2 → checkDaemon → checkRootUser → checkTargetDir (calls checkSudo if needed) → checkDockerGroup (calls checkSudo if needed). Fail-fast: return on first non-nil error from a hard-block check.

    Each check appends a CheckResult to a results slice before returning. CheckResult.Status is "pass", "warn" (for CHECK-03 and CHECK-07), or "fail".

    Every remote command uses a separate `client.NewSession()` call; close each session after use.
    Use `filetransfer.ShellQuote()` for all path and username arguments.
    All error and warning messages use `fmt.Fprintf(os.Stderr, ...)`.
    Pass ctx context.Context through but do not enforce a per-check deadline in Phase 5 (future wiring point).

    For tests: use a mock SSH client interface or a fakeSession approach that captures the command string and returns configurable exit code + stdout. Alternatively, define a narrow `sshSession` interface with `Output(cmd) ([]byte, error)` and `Run(cmd) error` so tests can inject fakes without a real SSH connection. The interface must match the gossh.Session method signatures exactly.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSH exec output → check logic | Remote host controls stdout/stderr of check commands; output is parsed to determine pass/fail |
| cfg.Host.User / cfg.Path → remote shell args | Values from deploy.yaml or --host flag are injected into SSH exec commands |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Injection | checkDockerGroup / checkTargetDir / checkSudo — cfg.Host.User and cfg.Path used in sudo commands | mitigate | ShellQuote() wraps all user-supplied values before shell concatenation; already pattern-established in Phase 3 |
| T-05-02-02 | Spoofing | checkDockerInstalled — parses `docker --version` stdout to confirm install | accept | Version string parsing is best-effort; the exit code (0/non-0) is the authoritative signal; output parsing is informational only |
| T-05-02-03 | Elevation of Privilege | checkSudo — grants passwordless sudo verification | accept | The check reads sudo access, does not grant it; sudo must already be configured by the operator |
</threat_model>
