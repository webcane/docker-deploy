    Create install.sh at the repo root. The script must be POSIX sh (#!/bin/sh, no bash-isms). Structure:

    VARIABLES section:
    - REPO: "webcane/docker-deploy"
    - INSTALL_DIR: "${HOME}/.docker/cli-plugins"
    - BINARY_NAME: "docker-deploy"
    - INSTALL_VERSION: use ${INSTALL_VERSION:-} to allow override; if empty, fetch latest from GitHub API: curl -fsSL https://api.github.com/repos/${REPO}/releases/latest | grep '"tag_name"' | sed 's/.*"tag_name": "\(.*\)".*/\1/'

    OS/ARCH DETECTION using uname:
    - OS: uname -s → map "Linux" → "linux", "Darwin" → "darwin"; exit 1 with message for unsupported OS
    - ARCH: uname -m → map "x86_64" → "amd64", "aarch64" or "arm64" → "arm64"; exit 1 with message for unsupported arch
    - ARCHIVE_NAME: "docker-deploy_${OS}_${ARCH}.tar.gz"
    - BASE_URL: "https://github.com/${REPO}/releases/download/${INSTALL_VERSION}"

    DOWNLOAD section:
    - Create a temp dir: TMPDIR=$(mktemp -d); trap 'rm -rf "$TMPDIR"' EXIT
    - Download the archive: curl -fsSL "${BASE_URL}/${ARCHIVE_NAME}" -o "${TMPDIR}/${ARCHIVE_NAME}"
    - Download checksums.txt: curl -fsSL "${BASE_URL}/checksums.txt" -o "${TMPDIR}/checksums.txt"

    SHA256 VERIFICATION (per D-15 — always verify, even without cosign):
    - Change to TMPDIR; use sha256sum if available, else shasum -a 256
    - Run: grep "${ARCHIVE_NAME}" checksums.txt | sha256sum -c - (or shasum equivalent)
    - If verification fails: print "ERROR: SHA256 checksum mismatch — aborting" and exit 1

    COSIGN VERIFICATION (per D-14 — attempt, fall back with warning):
    - Check if cosign is available: command -v cosign >/dev/null 2>&1
    - If cosign available: download checksums.txt.sig and checksums.txt.pem from same release URL; run: cosign verify-blob --certificate checksums.txt.pem --signature checksums.txt.sig checksums.txt
    - If cosign not available: print exactly: "cosign not found — skipping signature verification, checking SHA256 only" (per CONTEXT.md specifics)
    - If cosign verification fails (exit non-zero): print "WARNING: cosign signature verification failed — binary may be tampered" and exit 1

    INSTALL section:
    - Extract binary: tar -xzf "${TMPDIR}/${ARCHIVE_NAME}" -C "${TMPDIR}"
    - Create install dir: mkdir -p "${INSTALL_DIR}"
    - Check for existing binary; if present, get old version: OLD_VERSION=$("${INSTALL_DIR}/${BINARY_NAME}" --version 2>/dev/null || echo "unknown")
    - Move binary: mv "${TMPDIR}/${BINARY_NAME}" "${INSTALL_DIR}/${BINARY_NAME}"
    - Set executable: chmod +x "${INSTALL_DIR}/${BINARY_NAME}"
    - If old binary existed: print "Updated docker-deploy ${OLD_VERSION} → ${INSTALL_VERSION}" (per D-13)
    - If new install: print "Installed docker-deploy ${INSTALL_VERSION} to ${INSTALL_DIR}/${BINARY_NAME}"
    - Print: "Run: docker deploy --help"

    The script must be silent-overwrite on upgrade (per D-13) — no confirmation prompt.
    Use set -e at the top so any failed command aborts with a non-zero exit.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User machine → GitHub Releases CDN | Binary downloaded over HTTPS; attacker could MITM if cert is compromised |
| User machine → raw.githubusercontent.com | Script itself downloaded over HTTPS; same MITM risk |
| install.sh → checksums.txt | Checksum file downloaded from same release; provides integrity guarantee if release wasn't tampered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-02-01 | Tampering | Binary download | mitigate | SHA256 always verified against checksums.txt (D-15); cosign signature optionally verified (D-14); script aborts on checksum mismatch |
| T-09-02-02 | Tampering | install.sh itself | accept | Script downloaded over HTTPS from raw.githubusercontent.com; GitHub controls the TLS cert; no additional signing of the script itself (consistent with industry practice for curl-pipe-sh patterns) |
| T-09-02-03 | Spoofing | cosign verification | accept | cosign is optional; SHA256 is always enforced as the floor; users who want full supply chain can install cosign separately |
| T-09-02-04 | Elevation of Privilege | install.sh writing to ~/.docker/cli-plugins | accept | Writes to user's home directory only, no sudo required; binary placed in user-space plugin directory |
| T-09-02-05 | Information Disclosure | curl | pipe | sh pattern | accept | Standard OSS install pattern; README warns about the risk and provides manual install as alternative (per D-16) |
</threat_model>
