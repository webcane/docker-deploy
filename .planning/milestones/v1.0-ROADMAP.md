## Overview

docker-deploy is a Docker CLI plugin (`docker deploy`) that deploys docker-compose projects to remote VPS instances over SSH/SFTP. It copies project files to a staging directory, atomically swaps them into place, runs `docker compose up -d`, and reports container health — all without requiring git on the remote.

This milestone covers the full v1.0 scope: plugin scaffolding, SSH transport, file copy with atomic staging, core deploy loop, pre-flight checks, health polling, integration tests, distribution, documentation, CI tooling, shell completions, CLI subcommands, SSH config alias resolution, healthcheck config format, and release tooling.

## Milestone Summary

**Key Decisions:**

- Use `plugin.Run()` from `github.com/docker/cli` — binary named `docker-deploy` in `~/.docker/cli-plugins/`
- Manual `Resolve(flags, file, defaults)` — Viper excluded due to flag-override ordering bugs
- SFTP staging-dir pattern (`/tmp/docker-deploy-<ts>`) — prevents partial-deploy state
- `knownhosts` verification required — tool copies .env files, InsecureIgnoreHostKey is unacceptable
- SSH dial timeout via goroutine+context (not `ClientConfig.Timeout` alone)
- Phase 6 (init wizard) promoted to backlog Phase 999.1 — feature scope too large for v1.0 timeline
- Static completions shipped in goreleaser tarballs (replaced dynamic completion approach)
- SudoCreds stores password as `[]byte` with `Zero()` wipe for safe memory handling
- Warning rollup: single rollup at end without `--verbose`; inline per-line with `--verbose`

**Issues Resolved During Milestone:**

- CI DinD SSH host key capture race — fixed with retry loop + connection deadline
- Homebrew symlink not created on install — fixed with post_install + sandbox_allowlist
- GoReleaser archive missing contrib files at root — fixed with strip_parent: true
- install.sh using `/main/` instead of `/master/` — repo has no main branch
- Node.js 20 deprecation warnings in CI — bumped GitHub Actions versions
- Goroutine leak in RunCompose tests — refactored mock server, fixed defer ordering
- First-deploy mv nesting bug — rm -rf target placeholder before rename

**Issues Deferred:**

- Init wizard (Phase 999.1) — interactive setup for first-time VPS users
- Keyring integration (Phase 14-03) — macOS Keychain sudo credential caching
- Terminal demo recording (Phase 16 Wave 2) — vhs/asciinema for README
- SSH `Include` directive support — not yet implemented in sshconfig parser
- File permissions not preserved during SFTP upload (WR-01) — executable scripts lose +x
- Include field in file filter (FILES-03 partial) — exclude-only model for v1.0

**Technical Debt Incurred:**

- WR-01: SFTP does not preserve Unix file permissions — scripts may lose +x on remote
- WR-02: WalkFiles called twice (count + upload) — minor TOCTOU race in file count display
- CR-01: Sudo password may appear in error chain via format string on post-auth failures
- Phase 09 UAT partial — Homebrew E2E smoke test requires live tagged release (manual step)
- Phase 02/07/16 VERIFICATION.md: `human_needed` — live SSH host required for full verification
