## v1 Requirements

### Plugin (PLUG)

- [x] **PLUG-01**: Binary named `docker-deploy` in `~/.docker/cli-plugins/` responds to `docker deploy` in the Docker CLI — Phase 1 ✓
- [x] **PLUG-02**: `docker deploy --help` displays command usage natively within Docker CLI help — Phase 1 ✓
- [x] **PLUG-03**: Plugin responds to `docker-cli-plugin-metadata` with valid metadata JSON — Phase 1 ✓

### Deploy (DEPLOY)

- [x] **DEPLOY-01**: User can deploy with `docker deploy --host ssh://user@host:port` — Phase 4 ✓
- [x] **DEPLOY-02**: Project files are copied to remote via SFTP (`github.com/pkg/sftp` over the existing SSH connection) — Phase 3 ✓
- [x] **DEPLOY-03**: Files are staged to a temp directory first (`/tmp/docker-deploy-<timestamp>`) via SFTP, then moved atomically on the remote — Phase 3 ✓
- [x] **DEPLOY-04**: `docker compose up -d` is executed on the remote via SSH after file copy — Phase 4 ✓
- [x] **DEPLOY-05**: Plugin exits with non-zero code if any step (pre-flight, copy, compose) fails — Phase 4 ✓
- [x] **DEPLOY-06**: Deploy output (compose up result) is streamed back to the local terminal — Phase 4 ✓
- [x] **DEPLOY-07**: File copy supports structured auth fallback: (1) direct copy, (2) passwordless sudo, (3) interactive sudo password prompt — Phase 3+13 ✓

### File Management (FILES)

- [x] **FILES-01**: Default include list: compose.yaml (or docker-compose.yml), .env, Makefile, README.md — Phase 3 ✓
- [x] **FILES-02**: Default exclude list: .git/, node_modules/, vendor/, *.log, .DS_Store, __pycache__/ — Phase 3+7 ✓ (expanded)
- [x] **FILES-03**: User can extend exclude list via deploy.yaml or --exclude flag (exclude-only model for v1.0; no Include field) — Phase 3 ✓

### Configuration (CFG)

- [x] **CFG-01**: `--host` flag accepts `ssh://user@host:port` format — Phase 2 ✓
- [x] **CFG-02**: `--path` flag overrides default remote target directory (default: `/opt/<project_name>`) — Phase 2 ✓
- [x] **CFG-03**: `deploy.yaml` in project root stores persistent config (host, path, include, exclude) — Phase 2 ✓
- [x] **CFG-04**: Flag values take precedence over deploy.yaml; deploy.yaml takes precedence over defaults — Phase 2 ✓
- [x] **CFG-05**: `deploy.yaml` schema is forward-compatible with future multi-target extension (v2) — Phase 2 ✓
- [x] **CFG-07**: `~/.docker/cli-plugins/deploy.yaml` acts as a global defaults layer — Phase 15 ✓ (loadGlobalConfig() in main.go)

### Pre-Flight Checks (CHECK)

- [x] **CHECK-01**: Verify Docker is installed on remote host — Phase 5 ✓
- [x] **CHECK-02**: Verify `docker compose` v2 plugin is installed — Phase 5 ✓
- [x] **CHECK-03**: Verify Docker daemon is running on remote — Phase 5 ✓ (warning, not blocking)
- [x] **CHECK-04**: Check if SSH user is in the `docker` group — Phase 5 ✓ (warning + sudo attempt)
- [x] **CHECK-05**: Check passwordless sudo access; emit warning if not configured — Phase 5+13 ✓ (unconditional check; sudo password sanitized from errors)
- [x] **CHECK-06**: Verify target directory is writable; attempt `sudo mkdir -p` and `sudo chown` to fix — Phase 5 ✓
- [x] **CHECK-07**: Warn (but do not block) if SSH user is root — Phase 5 ✓

### Post-Deploy Health (HEALTH)

- [x] **HEALTH-01**: After `docker compose up -d`, poll container health status via `docker inspect` — Phase 5 ✓
- [x] **HEALTH-02**: Report final health result: pass (healthy) / fail (unhealthy) / unknown (no healthcheck) — Phase 5 ✓
- [x] **HEALTH-03**: Exit with non-zero code if any container reaches `unhealthy` state — Phase 5 ✓

## Out of Scope

| Feature | Reason |
|---------|--------|
| Docker remote context setup | Adds local complexity; observability via SSH exec is sufficient for v1 |
| Container registry / image push | Tool is file-copy-based; image building is user's responsibility |
| Blue-green / zero-downtime | Requires reverse proxy in compose.yaml — user-controlled |
| Rollback | Out of scope for v1; atomic staging-dir pattern limits blast radius |
| Git on VPS | Explicit non-requirement — the entire point of the tool |
| docker-compose v1 compatibility | EOL since June 2023; v1 is a hard-fail pre-flight condition |
| Web UI / server daemon | CLI-only; server-side complexity is a different product |
| Secrets vault integration | Out of scope; .env copied as-is |
| SSH Include directive | sshconfig parser does not follow Include directives (deferred) |
| File permissions preservation | SFTP does not preserve Unix permissions; entrypoint scripts may lose +x (WR-01 tech debt) |
