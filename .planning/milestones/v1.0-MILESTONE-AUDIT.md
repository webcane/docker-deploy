## Cross-Phase Integration

**Overall verdict: PASS with 3 warnings**

The E2E deploy flow (`--host` → config.Resolve → SSH dial → preflight → Upload → RunCompose → PollHealth) is fully wired. No broken connections exist between phases.

### E2E Flow Assessment

| Step | Status | Details |
|------|--------|---------|
| 1. Invoke `docker deploy --host` | WIRED | buildDeployCmd() registers --host, RunE dispatches to runDeploy() |
| 2. config.LoadFile + config.Resolve → Config | WIRED | Four-tier precedence; all 3 Resolve() call sites pass globalCfg (CFG-07) |
| 3. sshpkg.Dial → *ssh.Client | WIRED | Goroutine+select timeout; knownhosts TOFU; alias resolution via LookupHost |
| 4. preflight.RunPreflightChecks → CHECK gates | WIRED | NewSSHRunner adapter; NewSession per command; ShellQuote at all injection points |
| 5. filetransfer.Upload → atomic staging | WIRED | SudoCreds single prompt; all 8 ops via execCmd closure; rollback uses SudoExec |
| 6. compose.RunCompose → compose up -d | WIRED | PTY/pipe routing; isValidComposeFilename allowlist; ShellQuote on remote path |
| 7. health.PollHealth → per-container polling | WIRED | failCount retries gate; HEALTH-03 non-zero exit wired through runDeploy |

### Integration Warnings

**W-1: Health polling zero-config contract** (affects REQ: HEALTH-01, HEALTH-02)
`config.Config.Healthcheck` zero value is documented as "skip polling entirely" but `PollHealth` polls with 1ms interval / 1s timeout instead of skipping. This is harmless for compose files without `HEALTHCHECK` instructions (containers immediately report `"no-healthcheck"`), but causes a 1-second timeout failure for services that have a `HEALTHCHECK` instruction and no healthcheck config in `deploy.yaml` or flags.

**W-2: FILES-03 include whitelist unimplemented** (affects REQ: FILES-03)
`TargetConfig` only has `Exclude []string`. There is no configurable include whitelist — `WalkFiles` uploads all non-excluded files. The "copy everything minus excludes" model is a deliberate architecture choice and satisfies the common case, but users cannot configure a strict allowlist.
